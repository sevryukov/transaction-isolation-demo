📋 Сценарии семинара
1. Грязное чтение (Dirty Read)

Демонстрация работы приложения с данными, которых "не существует".

  • А: Начать транзакцию ➔ Добавить +500
  • (На сервере появился X-Lock)
  • Б: Изоляция ➔ Read Uncommitted
  • Б: Начать транзакцию ➔ Прочитать. (Видит 1500)
  • А: Rollback (Отмена действий).
  • Итог: Клиент Б прочитал данные (1500), которых в итоге нет на диске.
2. Защита от грязного чтения

Как работает Read Committed (по умолчанию в MS SQL).

  • А: Начать транзакцию ➔ Добавить +500
  • Б: Изоляция ➔ Read Committed
  • Б: Начать транзакцию ➔ Прочитать.
  • (Б зависает: S-Lock несовместим с чужим X-Lock)
  • А: Commit.
  • Итог: Клиент Б отвисает и читает консистентные 1500.
3. Неповторяющееся чтение

Изменение данных другой транзакцией "под ногами".

  • (Баланс 1000)
  • Б: Изоляция ➔ Read Committed
  • Б: Начать транзакцию ➔ Прочитать. (Видит 1000)
  • А: Начать ➔ Добавить +500 ➔ Commit.
  • Б: В той же транзакции ➔ Прочитать.
  • Итог: Б видит 1500! В рамках одной транзакции Б получил разные данные.
4. Защита (Repeatable Read)

Удержание S-блокировок для защиты строк.

  • (Баланс 1000)
  • Б: Изоляция ➔ Repeatable Read
  • Б: Начать транзакцию ➔ Прочитать. (На сервере повис S-Lock)
  • А: Начать ➔ Добавить +500.
  • (А зависает: X-Lock бьется об S-Lock)
  • Б: Снова прочитать (видит те же 1000) ➔ Commit.
  • Итог: А отвисает и выполняет апдейт.

Можно нажать "Rollback" у зависшей сессии для имитации отмены (Timeout).
