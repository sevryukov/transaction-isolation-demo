# Transaction Isolation Demo

This repository contains a simple demonstration of transaction isolation concepts commonly found in relational databases such as PostgreSQL, MySQL, and SQL Server. It currently includes an `index.html` file as a placeholder for content or examples.

## Overview

- **Purpose:** Provide an educational example or starting point for exploring transaction isolation levels and behaviors in database systems.
- **Contents:**
  - `index.html` - a basic HTML file serving as the project's main entry point.
  - `scripts/` - SQL scripts demonstrating isolation levels.
  - `LICENSE` - project license information.

## Features

- Demonstrates how different isolation levels (Read Uncommitted, Read Committed, Repeatable Read, Serializable) affect concurrency.
- Includes sample queries, diagrams or animations within `index.html` (to be expanded by contributors).
- Provides `scripts/` directory containing SQL examples for each isolation level.
- Extensible structure for adding code examples or interactive demos.

## Getting Started

1. Clone the repository:
   ```bash
   git clone https://github.com/sevryukov/transaction-isolation-demo.git
   cd transaction-isolation-demo
   ```
2. Open `index.html` in a web browser or serve it with a simple HTTP server (e.g. `python -m http.server`).
3. Browse through the examples and adjust the content or add new scenarios as needed.
